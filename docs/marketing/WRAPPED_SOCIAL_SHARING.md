# Wrapped Public Sharing Spec

**Status:** Ratified (2026-04-21)
**Owner:** marketing + platform
**Related:** L15-07, existing
`supabase/functions/generate-wrapped/`,
`omni_runner/lib/presentation/screens/wrapped_screen.dart`,
L04-15 (data portability), L04-12 (PII masking).

## Question being answered

> "Wrapped (Spotify-style year-in-review) is generated by an
> Edge Function and rendered inside the app, but there's no
> public URL like `/wrapped/[user]/2026` with an OG image
> that the user can paste into Twitter/Instagram. The
> viral-loop opportunity is missed."

## Decision

**Public, opt-in, signed-slug page** at
`/wrapped/[year]/[slug]` with a static OG image.

### URL shape

`https://omnirunner.com/wrapped/2026/{slug}`

`slug` is a 12-character base32 hash derived from
`(user_id, year, salt)`. NOT the user_id (so the URL doesn't
leak it; no enumeration). Same `(user_id, year)` always
resolves to the same slug — re-shareable without rotation,
revocable if needed (per-user salt rotation).

### Opt-in flow

1. User opens Wrapped in the app.
2. New section at the bottom: **"Compartilhar publicamente"**
   with a clear toggle.
3. Toggling ON writes a row to `wrapped_publications`:

```sql
CREATE TABLE public.wrapped_publications (
  user_id     uuid NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  year        smallint NOT NULL,
  slug        text NOT NULL UNIQUE,
  consent_at  timestamptz NOT NULL DEFAULT now(),
  revoked_at  timestamptz,
  view_count  integer NOT NULL DEFAULT 0,
  PRIMARY KEY (user_id, year)
);

ALTER TABLE public.wrapped_publications ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.wrapped_publications FORCE  ROW LEVEL SECURITY;

CREATE POLICY wrapped_self ON public.wrapped_publications
  FOR ALL TO authenticated
  USING (user_id = auth.uid())
  WITH CHECK (user_id = auth.uid());
```

4. Toggling OFF sets `revoked_at`. Public page returns 410
   Gone for revoked slugs (the cached OG image at the CDN
   gets stale-while-revalidated within 60 s).

### What the public page shows

- **Display name** (configurable; user can pick a nickname
  different from their account name in the consent step).
- **Total distance** for the year.
- **Total runs** for the year.
- **Top 3 longest runs** (date + distance + pace, no GPS
  trace).
- **Favourite running window** (morning / afternoon / evening).
- **Streak record** (longest consecutive days running).

Excluded by design:

- City / GPS coordinates (location PII).
- Running buddies / coach name (cross-tenant PII).
- Wallet / OmniCoins / payments (financial PII).
- Heart-rate / HRV / sleep (health PII per L04-04).

### OG image

Statically generated at publish-time by Vercel OG
(`/api/og/wrapped/[year]/[slug]`) and cached on the CDN with
`s-maxage=86400, stale-while-revalidate=604800`. Image is
1200×630 PNG with the same data as the page (no PII beyond
display name + aggregate numbers).

`<meta>` tags on the public page:

```html
<meta property="og:title" content="Meu Wrapped 2026 — Omni Runner" />
<meta property="og:image" content="https://omnirunner.com/api/og/wrapped/2026/{slug}" />
<meta property="og:url" content="https://omnirunner.com/wrapped/2026/{slug}" />
<meta property="twitter:card" content="summary_large_image" />
```

### LGPD / consent posture

- Consent registered in `consent_grants` (L04-03 envelope)
  with `policy_id='wrapped_public_share_v1'`.
- Revocation is one tap; CDN purge is best-effort but the
  HTML page returns 410 immediately.
- Once revoked, `view_count` and analytics about the public
  page are also purged (within 7 d via cron).

## Why this is in Wave 3, not now

- Vercel OG infra needs a font-loading pipeline + brand
  asset commit.
- Consent text needs legal review (it's a transfer to a
  potentially indeterminate audience).
- The slug rotation infrastructure for revocation + CDN purge
  is non-trivial.

Closing this finding now means the **shape is locked** and
the build is broken into clean Wave-3 deliverables.
